INDICATOR DEVELOPER DOCUMENTATION
Custom JavaScript Charting Platform

THIS IS NOT TRADINGVIEW PINESCRIPT.
Use plain JavaScript only. Do not use PineScript syntax or built-in TA helpers.

1) EXECUTION ENVIRONMENT
Indicator code runs as the BODY of an async function.

Allowed:
- Top-level statements
- Local helper functions
- Plain JS loops, arrays, Math
- await only for backtest()

Forbidden:
- function wrappers around the whole script
- import / export / require
- fetch / XMLHttpRequest / WebSocket
- DOM APIs (document, window, localStorage, etc.)
- setTimeout / setInterval
- external libraries
- assuming variables persist between runs

2) INPUT DATA
bars is a plain chronological OHLCV array:
- bars[i].time, open, high, low, close, volume
- Use manual loops for all calculations

bars has no TA methods.
Do not use:
- bars.close
- bars.sma(), bars.ema(), bars.rsi()
- bars.map(...).sma(), etc.

3) CFG RULE
cfg does not exist in normal indicator code.
Using it outside backtest() throws ReferenceError.

Normal indicator code:
- use fixed numbers directly

cfg exists only:
- inside the backtest strategy string
- after backtest() if you assign it yourself, e.g.:
  const cfg = result.bestParams ?? { length: 14 };

4) PLOT FUNCTIONS
plot(label, data, opts)       // line
plotHist(label, data, opts)   // histogram
plotBand(label, upper, lower, opts) // band fill
plotDot(label, data, opts)     // dots
plotArea(label, data, opts)    // filled line
plotCandle(label, data, opts)  // custom candles
plotLabel(label, data, opts)   // text/markers

Common opts:
{
  color: string,
  lineWidth: number,
  pane: number
}

Pane guide:
- pane 0: price overlays (MAs, bands, channels, candles)
- pane 1: oscillators, MACD, RSI, volume
- pane 2+: extra panes

All plotted values must be numbers.
Bad: { value: true }, { value: 'green' }
Good: { value: 42.5 }

5) TRADE FUNCTIONS
buy(time, price)  // entry marker / trade entry
sell(time, price) // exit marker / trade exit

Use bar timestamps and numeric prices.

6) BACKTESTING
Only use backtest() when explicitly requested for:
- strategy optimization
- P&L simulation
- parameter sweeps / testing combinations

Signature:
const result = await backtest({
  strategy: `...`,
  params: {
    length: { min: 5, max: 100, step: 1 },
    period: [5, 10, 14]
  },
  fees: {
    type: 'percent' | 'fixed',
    value: 0.1,
    min: null,
    max: 5
  },
  workers: 4
});

Result fields:
- result.bestParams
- result.bestScore
- result.bestTradeCount
- result.totalCombinations
- result.fees

Inside the strategy string:
- available: bars, cfg, buy(), sell()
- no plot() calls
- no await

After backtest():
- define cfg manually from result.bestParams if needed

Fee helper:
const entryCost = calcFee(entryPrice, fees);
const exitCost  = calcFee(exitPrice, fees);

7) COMMON EXAMPLES

SMA:
const len = 20;
const sma = [];
for (let i = len - 1; i < bars.length; i++) {
  let sum = 0;
  for (let j = i - len + 1; j <= i; j++) sum += bars[j].close;
  sma.push({ time: bars[i].time, value: sum / len });
}
plot('SMA 20', sma, { color: '#f59e0b', lineWidth: 2, pane: 0 });

EMA helper:
function ema(src, len) {
  const out = [];
  const k = 2 / (len + 1);
  let val = src[0];
  for (let i = 0; i < src.length; i++) {
    val = i === 0 ? src[0] : src[i] * k + val * (1 - k);
    out.push(val);
  }
  return out;
}

RSI:
- compute gains/losses manually
- plot on pane 1

MACD:
- EMA(12) - EMA(26)
- signal = EMA(9)
- histogram = MACD - signal
- plot lines + hist on pane 1

Bollinger Bands:
- mean +/− stddev over length
- use plotBand() on pane 0

Backtest pattern:
const STRATEGY = `... using cfg ...`;
const result = await backtest({
  strategy: STRATEGY,
  params: { length: { min: 5, max: 100, step: 1 } },
  fees: { type: 'percent', value: 0.1 },
  workers: 4
});
const cfg = result.bestParams ?? { length: 21 };

8) FORBIDDEN PATTERNS
Never write:
- cfg.anything outside backtest()
- bars.close or bars.* TA methods
- import / export / require
- fetch / DOM / timers
- string values in plotted data
- async wrappers around the whole indicator

Only available identifiers:
bars, plot, plotHist, plotBand, plotDot, plotArea, plotCandle, plotLabel, buy, sell, backtest, calcFee

Return only runnable indicator code in one fenced block:


No prose before or after.
